# Справочник алхимика — сайт

Небольшой сайт-проект (статические страницы) “Справочник алхимика”.

## Быстрый старт

### Вариант 1 — просто открыть в браузере
Если сайт состоит только из статических файлов (HTML/CSS/JS), можно открыть `index.html` двойным кликом.

### Вариант 2 — запустить локальный сервер (рекомендуется)
Так корректнее работают пути к файлам и запросы из JavaScript.

**Python 3:**

```bash
python -m http.server 8000
```

Потом открой `http://localhost:8000`.

**Node.js (если установлен):**

```bash
npx serve .
```

## Структура проекта

- `index.html` — входная страница
- `styles.css` — стили
- `main.js` — скрипты (подгружает рецепты из `recipes.json`)
- `recipes.json` — **единый список рецептов** для сайта и бота
- `bot.py` — простой Telegram-бот (приветствие + топ-рецепты из `recipes.json`)
- `requirements.txt` — зависимости для бота
- `assets/` — картинки/шрифты/иконки (если есть)

Редактируй рецепты только в **`recipes.json`** — после сохранения обнови страницу сайта (Ctrl+F5) и перезапусти бота, если он уже был запущен.

## Telegram-бот (MVP)

Нужны: аккаунт Telegram, приложение [@BotFather](https://t.me/BotFather) для создания бота и получения токена.

**1.** Установи зависимости:

```powershell
py -m pip install -r requirements.txt
```

**2.** Задай токен и (по желанию) ссылку на сайт после деплоя:

```powershell
$env:TELEGRAM_BOT_TOKEN="вставь_сюда_токен_от_BotFather"
$env:SITE_URL="https://кто-то.github.io/репозиторий"
```

Если `SITE_URL` не задан — бот всё равно ответит на `/start`, просто без кнопки «Открыть книгу».

**3.** Запуск:

```powershell
py bot.py
```

Остановить бота: **Ctrl+C**.

**Важно:** кнопка «Открыть книгу» должна вести на **публичный** адрес (например GitHub Pages). Адрес `http://127.0.0.1:8000` с телефона не откроется — это только твой компьютер.

## Разработка

- Редактируй файлы и обновляй страницу в браузере.
- Если пользуешься VS Code / Cursor, удобный вариант — расширение “Live Server”.

## Деплой

Самый простой вариант — **GitHub Pages**:

1. `Settings` → `Pages`
2. `Build and deployment` → выбрать ветку `main` и папку `/ (root)`
3. Сохранить — GitHub выдаст ссылку на сайт
